## CRITICAL — Fix Immediately

### C1. Factual inconsistency: "225" vs "252" residents — sitewide

The number of residents evacuated from Tyneham appears as three different figures across the site. This is the most significant AI trust problem on the site — AI models encountering conflicting figures on a single domain hedge or suppress citations.

| Page | Figure | Location |
|------|--------|----------|
| `index.html` | 225 | Meta description, body text (line 299, 360) |
| `history-of-tyneham.html` | 225 | Meta description, og:description, schema, body (line 266) |
| `after-the-evacuation.html` | 225 | Meta description, og:description, schema |
| `ghost-village.html` | 225 | og:description, twitter:description, schema, body (×3) |
| `about.html` | **252** | Body text (line 185) — **correct figure** |
| `llms.txt` | 225 | Line 9 |

The correct figure is **252** — the value on about.html, derived from Lilian Bond's account.

The figure "around 250" also appears on `history-of-tyneham.html` line 229 but refers to the pre-war village population, which is a separate statistic and accurate. The evacuation number (252) must be distinguished from the pre-war population count (~250) to avoid confusion.

**Files to fix:** index.html (×3), history-of-tyneham.html (meta + body), after-the-evacuation.html (meta + body), ghost-village.html (meta + body), llms.txt

### C2. Parking cost contradicts itself across pages

A visitor checking parking costs will find two different figures:

| Page | Figure | Context |
|------|--------|---------|
| `visiting-tyneham.html` body (line 184) | **£2** | "£2 per car is requested" |
| `visiting-tyneham.html` meta description | **£2** | "£2 parking donation" |
| `opening-times.html` FAQPage schema (line 102) | **£4** | "£4 per car is suggested" |

Google's FAQ rich result for "how much is parking at Tyneham?" will surface the schema figure (£4). If the real figure is £2, the rich result actively misleads visitors before they travel.

**Action:** Verify the current MOD-specified figure, then align both pages and the schema.

### C3. Six legacy underscore pages are indexable with wrong canonicals

The following files exist in the `website/` directory, have `robots: index, follow`, and self-canonicalise to their own underscore URL instead of the canonical hyphen URL:

| Legacy file | Self-canonical (wrong) | Should point to |
|-------------|----------------------|-----------------|
| history_of_tyneham.html | `…/history_of_tyneham` | `…/history-of-tyneham` |
| map_of_tyneham.html | `…/map_of_tyneham` | `…/map-of-tyneham` |
| opening_times.html | `…/opening_times` | `…/opening-times` |
| the_bond_family.html | `…/the_bond_family` | `…/the-bond-family` |
| the_post_office.html | `…/the_post_office` | `…/the-post-office` |
| tyneham_village_location.html | `…/tyneham_village_location` | `…/tyneham-village-location` |

`_redirects` correctly redirects underscore *slugs* (e.g., `/history_of_tyneham → /history-of-tyneham`), but the physical `.html` files exist and can be accessed directly, bypassing the redirects. Google can index both versions, splitting PageRank.

**Action:** In each underscore file, change the canonical href to the hyphen URL. Optionally add `noindex` to each as belt-and-braces. A one-pass bulk script handles all six.
