## HIGH

### 1. Three good pages are noindex — Google cannot index them

These pages are set to `noindex, nofollow` but contain substantial, quality content:

| Page | Words | noindex set in |
|---|---|---|
| `tyneham-in-wartime.html` | 1,145 | `<meta name="robots" content="noindex, nofollow">` |
| `corfe-castle-walk.html` | 899 | same |
| `kimmeridge-tyneham-walk.html` | 1,250 | same |

All three are well-structured articles with good H2s and schema. Unless they are deliberate drafts not ready for indexing, remove the robots meta tag or change to `index, follow`.

`tyneham-in-wartime.html` is particularly strong — it covers the WAAF radar station story which appears nowhere else online.

**Fix:** Remove the noindex meta tag from all three (or from whichever are actually live content).

## MEDIUM

### 2. `the-post-office.html` — H1 only, no H2 or H3

637 words, good content, but the entire page is one unbroken block under the H1. Search engines and readers rely on H2s to understand what a page covers.

**Current structure:**
```
h1: The Post Office
(637 words with no subheadings)
```

**Suggested structure:**
```
h1: The Post Office
  h2: Post Office Row
  h2: Inside the post office
  h2: After 1943
  h2: Visiting today
```

### 3. `the-rectory.html` — H3 used before H2 (hierarchy skip)

The content section "The Rectory at War" is wrapped in an `<h3>` but there is no `<h2>` above it. The only `<h2>` on the page is the WordPress comment count heading ("1 thought on 'The Rectory'"), which is not a real content heading.

**Current structure:**
```
h1: The Rectory
  h3: The Rectory at War   ← should be h2
h2: 1 thought on "The Rectory"  ← comment count, not content
```

**Fix:** Wrap content sections in `<h2>` tags. Also consider adding more sections — the page could cover The Rectory today, the 1966 fire, and architectural details.

### 4. `tyneham-remembered.html` — `<br>` inside H1 causes concatenated text

The H1 tag contains a line break tag:
```html
<h1>The Real Story of Tyneham<br>Told by the Villagers Themselves</h1>
```

When parsed by search engines and scrapers, this reads as: **"The Real Story of TynehamTold by the Villagers Themselves"** (no space). Use a space or CSS for the visual line break instead.

**Fix:**
```html
<h1>The Real Story of Tyneham: Told by the Villagers Themselves</h1>
```
Or use CSS `word-break`/`max-width` to wrap the heading visually.

### 5. Comment count headings as H2 dilute topic signals

Several pages use H2 tags for WordPress-style comment counts:
- `the-bond-family.html` → H2: "7 thoughts on 'The Bond Family of Tyneham'"
- `tyneham-house.html` → H2: "10 thoughts on 'Tyneham House'"
- `tyneham-church.html` → H2: "3 thoughts on 'Tyneham Church'"
- `tyneham-school.html` → H2: "6 thoughts on 'Tyneham School'"
- `history-of-tyneham.html` → H2: "3 thoughts on 'Tyneham Village History'"
- `tyneham-church.html`, `tyneham-house.html`, others

These headings add nothing topically and appear in the heading outline that search engines use to understand page content. Demote to `<h3>`, `<p>`, or a `<div>` with appropriate CSS styling.

### 6. No semantic `<main>` element on any page

Content is in a `<div class="col-lg-8">` but there is no `<main>` landmark element. This affects:
- **Accessibility** — screen readers use `<main>` to skip navigation
- **AI citability** — AI crawlers use landmarks to extract passage-level content
- **Structured data** — some parsers rely on `<main>` to scope content

**Fix:** Wrap the `col-lg-8` content div in a `<main>` tag (as a semantic wrapper, not replacing the div). This is a sitewide bulk edit.

### 7. `map-of-tyneham.html` — 113 words, no H2

The page is essentially just an embedded map with minimal surrounding text. It has H3s but no H2s above them (another hierarchy skip), and virtually no descriptive content.

**Current:** 113 words, `h1` → `h3` (skip)  
**Opportunity:** Add 200–300 words explaining the map, key landmarks, distances, and how to use it. Add H2 sections: "Key buildings on the map", "How to get to Tyneham".

### 8. `tyneham-remembered.html` — keyword meta tag

This page has a `<meta name="keywords">` tag with 12 comma-separated keywords:
```
Tyneham Remembered documentary, Tyneham DVD, Tyneham digital download, ...
```

Google has ignored `keywords` meta tags since 2009. Bing uses them as a spam signal. Remove it.

## LOW

### 9. `contact.html` — no H2 (acceptable, but worth adding)

The contact form page has 44 words and only an H1. Not a ranking page, but adding a brief H2 ("Send a message", "Other ways to get in touch") would improve structure and give users a clearer navigation landmark.

### 10. Gravatar comment avatars on static pages

Multiple pages embed `<img src="https://secure.gravatar.com/avatar/...">` images — these are WordPress comment section avatar images baked into the static HTML. On the live site they load from Gravatar's CDN; locally they appear broken.

**Pages affected:** `index.html` (25 avatars), `tyneham-house.html` (10), `tyneham-school.html` (5), `tyneham-church.html` (3), `history-of-tyneham.html` (3), `the-bond-family.html` (6), `the-rectory.html` (1).

These are benign in production but add HTTP requests and could slow page load. If comments are no longer being added, consider replacing gravatar `<img>` tags with a default SVG icon to reduce external requests.

### 11. `the-bond-family.html` — absolute URL for book cover image

One image uses an absolute production URL:
```html
<img src="https://tynehamvillage.org/assets/tyneham_a_lost_heritage.webp">
```
All other images use relative paths. This means if the domain changes or the site is tested locally, this image will fail. Change to: `assets/tyneham_a_lost_heritage.webp`
