# SEO Audit — tynehamvillage.org
**Date:** 2026-07-19
**Method:** Local file analysis (static HTML, 35 pages + assets/config)
**Previous audit:** 2026-07-13

---

## Executive Summary

**Health: 82/100** — up from ~78 in the July 13 audit.

The site has **strong technical foundations**: clean URLs, comprehensive redirects, good security headers, all AI crawlers allowed, HSTS preloaded, proper caching. All 35 content pages have title/description/OG/Twitter/canonical tags. Schema coverage is 34/35 pages (only privacy missing).

**Top 3 issues carrying over from July 13 (still unfixed):**
1. Privacy page missing from sitemap and has no JSON-LD schema
2. 271 orphaned images bloating the repo
3. Only 15/35 pages use `<picture>` with WebP — the rest deliver raw JPEGs

**New findings this audit:**
- Title-H1 misalignment on 7 pages
- No favicon on 33/35 pages
- FAQPage schema only on homepage (missed opportunity for opening-times)
- Excessive internal link volume to `/tyneham-remembered/` (181 links — 3x any other page)

## 2. On-Page SEO

### 2.1 Title Tags ✓
- All 36 pages have unique `<title>` tags
- All include primary keyword + brand modifier
- Lengths: 50–73 characters (good — visible in SERP)
- Well-crafted, click-worthy

### 2.2 Meta Descriptions ✓
- All 35 content pages have descriptions
- Typically 150–160 characters
- Include primary keywords and value propositions

### 2.3 Title-H1 Alignment — 7 PAGES WITH MISMATCHING KEYWORDS
Pages where `<title>` and `<h1>` differ in keyword selection:

| Page | Title | H1 | Impact |
|------|-------|----|--------|
| `camp-at-tyneham` | "Camping near Tyneham Village" | "Can You Camp at Tyneham?" | Medium |
| `index.html` | "Tyneham Village: Dorset's Ghost Village" | "Tyneham Village in Dorset" | Medium |
| `visiting-tyneham` | "Visiting Tyneham" | "Visiting Tyneham Village" | Low |
| `flowers-barrow` | "Flowers Barrow" (no apostrophe) | "Flower's Barrow" (with apostrophe) | Medium |
| `map-of-tyneham` | "Tyneham Map" | "Tyneham Village Map" | Low |
| `the-church-door-note` | "The Note on the Church Door – Tyneham Village" | "The Note on the Church Door" | Low |
| `tyneham-remembered` | "Tyneham Remembered: Village Documentary DVD & Download" | (multi-line heading) | Low |

**Recommendation:** Align h1 with the primary keyword in the title. Google uses both to understand page topic. Mismatches weaken topical focus.

### 2.4 Heading Hierarchy — MIXED

**Good (proper H1→H2→H3 structure):**
- `index.html` (h2=11, h3=7), `ghost-village` (h2=8), `tyneham-house` (h2=4, h3=3), `durdle-door` (h2=8, h3=6)

**Problematic (h3 used without sufficient h2 context):**
- `opening-times`: h2=1, h3=5 — single h2, all subsections are h3 (low structural depth)
- `tyneham-school`: h2=1, h3=3 — same issue
- `tyneham-church`: h2=4, h3=1 — only 1 subheading at h3 level

**Thin heading structure:**
- `map-of-tyneham` (h2=2, h3=0, ~1,730 words)
- `contact` (h2=1, h3=0, ~1,696 words)
- `thank-you` (h2=0, h3=0, ~765 words)

**Recommendation:** Ensure headings follow H1→H2→H3 hierarchy. Thin pages need more content or better structure.

### 2.5 Image Optimization — BELOW AVERAGE

**Alt text:** ✓ All images have alt attributes. No empty alt text found.

**WebP usage:** Only 15/35 pages use `<picture>` with WebP source + JPG fallback. The remaining 20 use plain `<img>` with .jpg — losing WebP's ~40% size reduction.

**Image preloads:** Only on 4 pages (index, ghost-village, history-of-tyneham, visiting-tyneham). Missing on opening-times.html (highest-traffic page).

**Social images:** Nearly all pages use the same OG image (`tyneham-village-early-photo.webp`). Should be page-specific for better click-through from social.

**Favicon:** Only `about/` and `404.html` declare a favicon. Missing from 33/35 content pages.

**Action:**
1. Add `link rel="icon"` to all page templates
2. Add `link rel="preload" as="image"` to opening-times and walk pages
3. Convert remaining `.jpg`-only `<img>` tags to `<picture>` with WebP + JPG

### 4.2 Caching ✓

- CSS: same as above
- JS: same as above

### 4.3 CSS Size — 193KB MINIFIED
The minified CSS (styles.min.css) is 193KB. For a static content site with Bootstrap loaded from CDN (~150KB), 193KB of custom CSS is substantial. The uncompressed version is 241KB.

**Recommendation:** Audit CSS for unused rules. Bootstrap CDN already provides most utility classes — custom CSS should be minimal.

### 4.4 Assets Directory — 44MB / 210 FILES
Up from the July 13 audit. 271 orphaned images (files in `/assets/` not referenced by any HTML page).

**Action:** Delete unused images to reduce repo/bandwidth. (Carried over from July 13 audit.)

|------|-----------------|-------|
| `/tyneham-remembered/` | **181** | DVD sales page — 3x any other page |
| `/opening-times/` | 159 | ✓ Appropriate (high-intent page) |

| `/tyneham-school/` | 125 | ✓ |
| `/the-bond-family/` | 121 | ✓ |
| `/history-of-tyneham/` | 119 | ✓ |

The `/tyneham-remembered/` page has 181 internal links — 3x the next-highest page. Most of these are sidebar menu links that appear on every page. While monetization is valid, Google may interpret excessive internal links as over-optimization.

**Recommendation:** Reduce sidebar prominence of the DVD link. Consider moving it to footer or a less linked position.
